# イメージ

[← 手順書に戻る](../README.md)

**概要:**
Docker イメージはコンテナ起動に必要なファイル、ライブラリ、設定をまとめた読み取り専用テンプレートです。イメージを基にコンテナが作られます。

**主なポイント:**

- ベースイメージを指定して Dockerfile で拡張するのが一般的。
- イメージにはタグを付けてバージョン管理する（再現性確保）。

**実務上の注意:**

- セキュリティパッチを定期的に適用した最新イメージを利用する。
- イメージサイズを抑えるために不要ファイルを削除するベストプラクティスを採用する。

**コマンド例:**

```
docker build -t myapp:1.0 .
docker pull wordpress:latest
```
