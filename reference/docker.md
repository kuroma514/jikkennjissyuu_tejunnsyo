# Docker

[← 手順書に戻る](../README.md)

**概要:**
Docker はアプリケーションと依存関係を「イメージ」としてパッケージし、それを実行する「コンテナ」で一貫して動作させる技術です。開発・テスト・デプロイの再現性を高めます。

**主なポイント:**

- イメージ: 実行可能なテンプレート。
- コンテナ: イメージの実行インスタンス。
- レジストリ: イメージの配布場所（Docker Hub 等）。

**実務上の注意:**

- 永続化はボリュームを使う（DB データ等）。
- セキュリティ: 機密情報は環境変数やシークレットで管理。
- 再現性: イメージには明示的タグを付ける。

**コマンド例:**

```
docker build -t myapp:1.0 .
docker run -d --name myapp -p 8080:80 myapp:1.0
```
