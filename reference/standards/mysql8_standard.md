# MySQL 8.0

[← 手順書に戻る](../3I-45-手順書.md)

**概要:**
MySQL 8.0 はモダンな機能とセキュリティ強化が入った RDBMS です。コンテナで利用する際は初期化や永続化、認証方式に注意して運用する必要があります。

**主なポイント:**

- 起動時の環境変数で初期データベースやユーザーを作成できる。
- データ永続化はボリュームを使う（例: `/var/lib/mysql`）。
- 認証プラグインの違いで接続エラーが起きることがある。

**実務上の注意:**

- バックアップと復旧手順を必ず用意する（`mysqldump` 等）。
- パフォーマンス系パラメータ（`innodb_buffer_pool_size` 等）はコンテナのメモリに合わせて調整する。

**コマンド例:**

```
docker compose logs --tail 100 db
```

**参考:** MySQL 公式ドキュメント
