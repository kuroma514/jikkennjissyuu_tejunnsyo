# ポートバインド／競合エラー

[← 手順書に戻る](../README.md)

**概要:**
ポートバインドエラー（例: `Bind for 0.0.0.0:8080 failed`）は、ホスト側のポートが他プロセスにより占有されている場合に表示されます。Docker 起動時にホストポートを指定する際は競合確認が必要です。

**主なポイント:**

- ホストでポートを使っているプロセスがあるとバインドできない。
- 既存コンテナや別アプリが占有していることが多い。

**対処法:**

- 別ポートを指定する（例: `8081:80`）。
- 使用プロセスを特定して停止する（Windows: `netstat -ano | findstr 8080` 等）。
- 既存のコンテナを停止/削除する。

**運用の注意:**

- 開発チームでポート割当ルールを決めておくと競合が減る。
- 本番はリバースプロキシ等で公開ポートを集中管理する。
