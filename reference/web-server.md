# Web サーバ

[← 手順書に戻る](../README.md)

**概要:**
Web サーバは HTTP リクエストを受け取り、静的ファイルや動的に生成したレスポンスを返すサーバソフトウェアです。代表例に Apache や Nginx があり、PHP 等と組み合わせて動的サイトを提供します。

**主なポイント:**

- 静的コンテンツ配信と動的処理の橋渡しを行う。
- 設定（VirtualHost/ServerBlock）で複数サイトを運用可能。
- コンテナ化すると設定やデプロイが再現しやすい。

**実務上の注意:**

- ログローテーションやアクセス制御（CORS・TLS）を適切に設定。
- コンテナではポート公開、ボリュームでドキュメントルートを管理する。

**例:**

```
# Docker で Apache を 8080 で公開
docker run -d -p 8080:80 httpd:latest
```

**参考:** Apache / Nginx 公式ドキュメント
