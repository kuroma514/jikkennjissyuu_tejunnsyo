# コンテナ

[← 手順書に戻る](../3I-45-手順書.md)

**概要:**
コンテナはアプリケーションとその依存を分離して実行する単位です。軽量で起動が速く、環境の再現性に優れています。Docker ではイメージからコンテナを作成して実行します。

**主なポイント:**

- イメージから作られ、実行中は分離されたプロセス空間で動作する。
- リソース制限（メモリ/CPU）やボリューム、ネットワーク設定が重要。

**実務上の注意:**

- ログの取り扱いや監視、再起動ポリシーを設定する。
- データ永続化はボリューム、構成管理はイメージと設定ファイルで行う。

**コマンド例:**

```
docker ps
docker logs <container>
```

**参考:** Docker 公式ドキュメント（コンテナ）
