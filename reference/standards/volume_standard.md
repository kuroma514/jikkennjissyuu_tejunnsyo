# Volume

[← 手順書に戻る](../3I-45-手順書.md)

**概要:**
Docker Volume はコンテナのデータを永続化する仕組みで、コンテナ再作成やアップデート後もデータを保持できます。データベースやアップロードファイルの永続化に使います。

**主なポイント:**

- 名前付きボリュームは Docker が管理するため移植性が高い。
- バインドマウントはホストディレクトリを直接マウントする。

**実務上の注意:**

- DB データはボリュームで保存し、ボリュームのバックアップ戦略を持つ。
- バインドマウントは開発で便利だがパーミッション差に注意。

**コマンド例:**

```
docker volume ls
docker volume inspect db_data
```

**参考:** Docker ボリュームのドキュメント
